Promise Events

None of this code will run, but give it a read so you know how to handle promise events.

.done()

.done() is raised when the operation completes successfully, and accepts one or more event handler functions (parameters).

promise.done(function(data) {

  // data returned is determined by the operation

});

.fail()

.fail() is raised when the operation has completed with an error. Like done, fail accepts one or more parameters.

promise.fail(function(data) {

  // data returned is determined by the operation

});

.progress()

.progress() is raised when the operation raises an alert about its current state. Not all operations raise progress events.

promise.progress(function(data) {

  // data returned is determined by the operation

});

Chaining promise events

You can add .done() and .fail() (and sometimes, when applicable, .progress()) event handlers by chaining the method calls as demonstrated below.

promise.done(function(data) {

  // success

}).fail(function(data) {

  // failure

});

.then()

.then() is a single function that allows you to register .done(), .fail(), and .progress() event handlers in one call. The sample below is identical to th echaining demonstration above.

promise.done(function(data) {

  // success

}).fail(function(data) {

  // failure

});
